# Especificação de Requisitos Funcionais - Sistema de Gestão de Livraria

## REQUISITOS

### REQUISITOS FUNCIONAIS

#### RF01. O sistema deve permitir a compra de livros.
Usuário: Cliente.
  
#### RF02. O sitema deve permitir o acesso de um cliente ao app mesmo sem estar logado.
Usuário: Cliente.

#### RF03. O sistema deve possuir um sistema de promoções.
Usuário: Administrador, Cliente.

#### RF04. O sistema deve exibir o relatório de vendas.
Usuário: Administrador.

#### RF05. O sistema deve ser capaz de armazenar livros dentro de um carrinho de compras.
Usuário: Cliente.

#### RF06. O sistema deve ser capaz de listar todos os livros disponíveis no catálogo.
Usuário: Cliente.

#### RF07.  Dentro do sistema deve ser possível avaliar os livros comprados anteriormente.
Usuário: Cliente, Administrador.

#### RF08.  Deve ser possível consultar o estoque disponível da livraria.
Usuário: Administrador.

#### RF09.  O sistema deve permitir a retirada de livros do estoque.
Usuário: Administrador.

#### RF10. O sistema deve permitir a reposição de livros no estoque.
Usuário: Administrador.

#### RF11.  O sistema deve permitir dois níveis de acesso diferentes, administrador e cliente.
Usuário: Administrador, Cliente.

#### RF12. O sistema deve permitir que novos livros possam ser adicionados ao catálogo do sistema.
Usuário: Administrador.

#### RF13. O sistema deve permitir a remoção de livros no catálogo do sistema.
Usuário: Administrador.

#### RF14. Deve ser possível excluir uma conta de cliente do sistema.
Usuário: Administrador, Cliente.

#### RF15. O sistema deve possuir uma interface simples contendo uma aba para a conta do cliente, uma aba para pesquisa de livros e a página principal.
Usuário: Cliente.

#### RF16. O sistema deve possuir uma parte dedicada para suporte, com um guia sobre as principais funcionalidades do site.
Usuário: Cliente.

#### RF17. O sistema deve permitir a visualização dos dados dos clientes, com exceção de senha e informações de cartão de crédito.
Usuário: Administrador.
